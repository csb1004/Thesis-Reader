# Thesis Reader 설계

## 목표

휴대폰에서 논문을 편하게 읽을 수 있는 Flutter 앱을 만든다. 사용자는 PDF 논문을 가져오고, 앱은 이를 카카오페이지처럼 읽기 쉬운 문서 패키지로 변환한 뒤 캐시에 저장한다. 이후 같은 논문을 열 때는 원본 PDF를 다시 분석하지 않고 캐시된 문서 패키지를 사용한다.

MVP는 안정적인 논문 읽기, 품질 우선 PDF 구조화, 그림/표/수식/각주/참고문헌 접근, OpenAI 기반 번역/요약, 논문별 단어장 관리를 우선한다.

## 핵심 결정

- 플랫폼: iOS와 Android를 지원하는 Flutter 앱.
- PDF 변환: Railway에 올린 품질 우선 변환 서버를 기본으로 사용.
- 변환 fallback: 서버 작업 생성이 10초 안에 실패하거나 서버 작업이 실패하면 앱 내부 변환으로 전환.
- 캐시 방식: 최초 1회 변환 후 로컬 문서 패키지로 저장하고, 이후에는 캐시된 패키지를 연다.
- 원본 PDF 보관: 앱 내부 저장소에 원본 PDF 사본을 복사해 관리.
- 서버 개인정보 정책: 서버는 PDF와 변환 결과를 작업 중에만 임시 보관한다. 앱이 결과를 내려받으면 삭제하고, 방치된 작업은 TTL cleanup으로 삭제한다.
- 계정: MVP에는 사용자 계정을 만들지 않는다. 서버 변환은 익명 작업 ID로 관리한다.
- 추후 동기화: stable document ID와 package version을 두어 나중에 계정 동기화를 붙일 수 있게 한다.
- OpenAI 사용: 사용자가 자기 OpenAI API key를 앱에 입력한다. 앱은 key를 안전 저장소에 보관하고 OpenAI를 직접 호출한다. Railway 서버는 OpenAI key, 선택 텍스트, 번역, 요약을 다루지 않는다.
- 번역 방향: MVP는 영어를 한국어로 번역한다.

## 아키텍처

시스템은 Flutter 앱, Railway 변환 서버, OpenAI API 세 부분으로 나뉜다.

### Flutter 앱

앱은 읽기 경험과 MVP의 모든 사용자 데이터를 담당한다.

- PDF 가져오기와 원본 PDF 로컬 복사.
- 문서 라이브러리.
- 변환 작업 생성과 상태 확인.
- 문서 패키지 캐시 저장.
- 페이지 모드와 스크롤 모드.
- 뷰어 설정.
- 읽던 위치 저장.
- 그림/표/수식/각주/참고문헌 링크 상호작용.
- 논문별 단어장.
- 섹션 요약 캐시.
- OpenAI API key 보안 저장.
- 번역, 단어 뜻, 요약을 위한 OpenAI 직접 호출.

### Railway 변환 서버

서버는 품질 우선 PDF 변환을 담당한다.

- 업로드된 PDF를 받아 익명 변환 작업을 만든다.
- 논문 구조를 최대한 추출한다.
- 앱 내부 fallback 변환기와 같은 문서 패키지 스키마를 출력한다.
- 작업 입력물과 결과물을 임시 저장한다.
- 앱이 작업 상태를 조회하고 결과를 내려받을 수 있게 한다.
- 앱 다운로드 완료 또는 TTL 만료 시 작업 데이터를 삭제한다.

서버는 PyMuPDF 같은 실용적인 PDF 도구를 우선 사용한다. 변환 품질이 부족하면 이후 GROBID 같은 더 무거운 논문 구조화 도구를 품질 개선 옵션으로 추가할 수 있다.

### OpenAI API

OpenAI는 앱에서 사용자의 API key로 직접 호출한다.

- 문맥 기반 단어 뜻.
- 선택한 구/문장 번역.
- 선택한 구간 요약.
- 섹션 전체 요약.

OpenAI 요청에는 현재 기능에 필요한 텍스트와 짧은 문맥만 보낸다.

## PDF 가져오기와 변환 흐름

1. 사용자가 PDF를 가져온다.
2. 앱이 PDF를 앱 내부 저장소에 복사한다.
3. 앱이 stable document ID를 가진 로컬 문서 레코드를 만든다.
4. 앱이 Railway 변환 작업 생성을 시도한다.
5. 작업 생성이 10초 안에 성공하지 않으면 앱 내부 변환을 시작한다.
6. 서버 작업이 시작되면 앱은 익명 job ID를 저장하고 상태를 주기적으로 확인한다.
7. 사용자가 앱을 백그라운드로 내려도 서버 변환은 계속 진행된다.
8. 변환 화면에는 진행 상태와 "원본 PDF로 먼저 보기" 버튼을 제공한다.
9. 변환이 끝나면 앱이 문서 패키지를 내려받는다.
10. 앱은 문서 패키지를 로컬 캐시에 저장하고 서버에 임시 데이터 삭제를 요청한다.
11. 이후 같은 문서를 열 때는 캐시된 문서 패키지를 사용한다.

서버 작업이 시작된 뒤 실패하면 앱은 서버 변환 실패 상태를 보여주고 앱 내부 변환으로 계속 진행한다.

## 백그라운드 변환

기본 변환은 서버에서 진행되므로 백그라운드 처리는 주로 Railway 작업으로 해결한다.

- 앱은 job ID를 로컬에 저장한다.
- 앱이 닫히거나 백그라운드에 있어도 서버 변환은 계속된다.
- 앱이 다시 열리면 job 상태를 확인하고 결과가 준비되어 있으면 내려받는다.

앱 내부 fallback 변환의 경우:

- Android는 가능한 범위에서 background task 또는 foreground notification으로 변환을 계속한다.
- iOS는 백그라운드 실행 제한이 있으므로 앱이 중단되면 재실행 시 이어서 처리하거나 다시 시작한다.

문서 패키지는 변환이 완료된 뒤에만 저장된다. 사용자가 읽는 중인 리더 화면은 자동으로 재구성하지 않는다.

## 문서 패키지 모델

각 논문은 변환 후 하나의 로컬 문서 패키지를 가진다. 문서 패키지는 다음 정보를 포함한다.

- `metadata`: 제목, 저자, 원본 파일명, 가져온 날짜, 원본 PDF hash, 변환기 버전, 패키지 버전.
- `sections`: 섹션 ID, 제목, 순서, 포함 block 범위.
- `blocks`: 제목, 문단, 인용, 그림, 표, 수식, 각주, 참고문헌 같은 순서 있는 읽기 단위.
- `assets`: 추출 이미지, 표 스냅샷, 수식 스냅샷, 원본 페이지 영역 스냅샷, 썸네일.
- `referenceSpans`: `Figure 2`, `Table 1`, 수식 번호, 각주 표시, 참고문헌 인용 같은 텍스트 범위.
- `anchors`: `blockId`, 텍스트 offset, 원본 PDF page, 원본 PDF 좌표를 포함하는 안정적인 읽기 위치.
- `vocabulary`: 논문별 단어장 항목.
- `summaries`: 선택적으로 저장하는 섹션 요약 캐시.

그림, 표, 수식, 각주, 참고문헌은 최대한 구조화한다. 추출 신뢰도가 낮으면 원본 PDF의 해당 페이지 영역을 이미지 스냅샷으로 보존해 내용이 사라지지 않게 한다.

## 리더 UX

리더는 페이지 모드와 스크롤 모드를 모두 지원한다.

### 페이지 모드

페이지 모드는 문서 패키지를 현재 화면 크기에 맞춰 한 화면 단위로 재조판한다. 페이지 나눔은 다음 설정의 영향을 받는다.

- 기기 화면 크기.
- 테마.
- 글꼴.
- 글자 크기.
- 줄 간격.
- 여백.

뷰어 설정을 바꾸면 페이지 경계는 달라질 수 있다. 하지만 block 순서, asset 연결, 단어장, 섹션 구조는 바뀌면 안 된다.

Android 볼륨키 이동:

- 볼륨 다운: 다음 페이지.
- 볼륨 업: 이전 페이지.

iOS는 플랫폼 제약 때문에 볼륨키 이동을 보장하지 않는다. iOS에서는 스와이프, 좌우 탭 영역, 화면 내 이동 컨트롤을 제공한다.

### 스크롤 모드

스크롤 모드는 같은 문서 block을 세로 흐름으로 보여준다. 페이지 모드와 같은 테마, 글꼴, 줄 간격, 여백, reference span, 번역, 단어장 동작을 사용한다.

### 뷰어 설정

설정 패널은 사용자가 보낸 카카오페이지 스타일을 따른다.

- 열람 방식: 페이지 또는 스크롤.
- 테마 프리셋.
- 글꼴: 시스템 기본, serif, sans.
- 글자 크기.
- 줄 간격.
- 여백.
- 그림/표 열기 방식: 하단 시트 또는 전체 화면.

MVP에서는 글꼴 프리셋을 작게 시작하고, 이후 필요하면 글꼴 카탈로그를 확장한다.

## Reference와 Asset 상호작용

Reference span은 본문에서 색을 다르게 표시해 사용자가 누를 수 있는 요소임을 알 수 있게 한다.

짧게 탭했을 때:

- 그림/표/수식/각주/참고문헌 span은 연결된 내용을 연다.
- 시각 자료는 설정에 따라 하단 시트 또는 전체 화면으로 열린다.

지원하는 시각 자료 열기 방식:

- 하단 시트: 읽던 위치를 유지하며 빠르게 그림, 표, 수식을 확인한다.
- 전체 화면 뷰어: 확대, 이동, 자세한 확인에 적합하다.

본문 안 확장은 MVP 범위에 포함하지 않는다.

길게 누르거나 드래그했을 때:

- 텍스트를 길게 누르면 번역 선택 동작을 시작한다.
- 길게 누른 상태로 드래그하면 구, 문장, 범위를 선택한다.
- 선택 범위에 reference span이 포함되어도, 드래그 선택 중에는 링크 열기보다 번역 선택을 우선한다.

## 번역, 요약, 단어장 UX

번역은 짧은 탭이 아니라 길게 누르기로 시작한다. 페이지 넘김, 스크롤, 링크 탭과 충돌하지 않게 하기 위해서다.

### 단어

사용자가 단어를 길게 누르면:

1. 앱이 단어와 원문 문장을 식별한다.
2. 앱이 앞뒤 문맥과 섹션 정보를 포함한다.
3. 앱이 OpenAI를 직접 호출한다.
4. 앱이 논문 문맥에 맞는 한국어 뜻을 보여준다.
5. 앱이 해당 논문의 단어장에 자동 저장한다.

### 구, 숙어, 문장

사용자가 길게 누른 뒤 드래그하면:

1. 앱이 구, 숙어, 문장, 또는 범위를 선택한다.
2. 앱이 선택 영역을 한국어로 번역한다.
3. 선택 영역이 짧은 구나 숙어라면 번역 시트에서 "단어장에 추가" 버튼을 제공한다.
4. 문장 전체나 긴 범위 번역은 자동으로 단어장에 저장하지 않는다.

### 요약

MVP는 다음 요약을 지원한다.

- 선택한 구간 요약.
- 섹션 전체 요약.

섹션 전체 요약은 섹션의 block들을 모으고, 토큰 제한에 맞춰 나누고, 필요하면 chunk별 요약 후 통합한다. 섹션 요약 결과는 문서와 섹션 단위로 캐시할 수 있다.

## 논문별 단어장

단어장은 가져온 논문별로 분리한다.

각 항목은 다음을 저장한다.

- 표현.
- 논문 문맥에 맞는 한국어 뜻.
- 원문 문장.
- 앞뒤 문맥.
- 논문 위치 anchor.
- 생성 날짜.
- 마지막 수정 날짜.
- 사용자가 수정한 뜻.
- 사용자 메모.

중복 방지는 같은 논문 안에서만 한다. normalization key는 표현을 소문자화하고, 공백을 정규화하고, 의미 없는 구두점을 제거한다. 이 방식은 단어뿐 아니라 숙어와 기술 구문 같은 여러 단어 표현도 지원한다.

사용자가 이미 저장된 표현을 다시 길게 누르면 새 항목을 만들지 않고 기존 단어장 항목을 열거나 업데이트한다.

## 읽던 위치

앱은 논문별 읽던 위치를 저장한다.

읽던 위치 anchor는 다음을 포함한다.

- `blockId`.
- block 내부 텍스트 offset.
- 원본 PDF page.
- 가능한 경우 원본 PDF 좌표.
- 리더 모드와 대략적인 시각 진행률.

사용자가 앱을 다시 열면 마지막으로 읽던 논문과 위치로 이동한다. 뷰어 설정이 바뀌어 페이지 경계가 달라졌다면 단순 페이지 번호가 아니라 anchor를 사용해 가장 가까운 텍스트 위치로 이동한다.

## 저장 정책

앱 로컬 저장소에는 다음을 저장한다.

- 원본 PDF 사본.
- 캐시된 문서 패키지.
- 추출된 asset.
- 뷰어 설정.
- 읽던 위치.
- 논문별 단어장.
- 섹션 요약 캐시.
- 안전 저장소에 저장된 OpenAI API key.

서버 저장소에는 변환 중인 임시 데이터만 둔다.

- 활성 작업의 업로드 PDF.
- 중간 변환 파일.
- 앱 다운로드를 기다리는 최종 패키지.

서버 cleanup 규칙:

- 앱 다운로드와 확인이 끝나면 삭제.
- 앱이 내려받지 않으면 TTL 이후 삭제.
- 실패한 작업 artifact도 TTL 이후 삭제.

## 오류 처리

변환 서버를 사용할 수 없을 때:

- 작업 생성이 실패하거나 10초를 넘기면 앱 내부 변환을 시작한다.

서버 작업이 실패했을 때:

- 실패 상태를 보여준다.
- 앱 내부 변환으로 계속 진행한다.
- 원본 PDF 보기는 계속 가능해야 한다.

앱 내부 변환도 실패했을 때:

- 원본 PDF 보기를 유지한다.
- 문서를 미변환 상태로 표시한다.
- 사용자가 나중에 서버 변환을 다시 시도할 수 있게 한다.

OpenAI key가 없을 때:

- 번역과 요약 동작을 비활성화한다.
- key 설정 안내를 보여준다.

OpenAI 요청이 실패했을 때:

- 선택 텍스트를 유지한다.
- 재시도와 복사 동작을 제공한다.
- 의미가 성공적으로 반환되지 않았다면 단어장 항목을 만들지 않는다.

추출 신뢰도가 낮을 때:

- 원본 페이지 영역 스냅샷으로 내용을 보존한다.
- 읽기 순서를 안정적으로 유지한다.
- 논문 내용을 조용히 누락하지 않는다.

## 테스트 계획

### PDF 가져오기와 변환

- 텍스트 기반 단일 컬럼 PDF 가져오기.
- 2단 컬럼 논문 가져오기.
- 그림이 많은 논문 가져오기.
- 표가 많은 논문 가져오기.
- 수식이 많은 논문 가져오기.
- 서버 변환 성공 확인.
- 서버 작업 생성 10초 timeout fallback 확인.
- 서버 작업 실패 fallback 확인.
- 변환 중 원본 PDF 보기 확인.
- 캐시된 패키지가 재분석 없이 열리는지 확인.

### 리더 동작

- 페이지 모드와 스크롤 모드 전환.
- 글자 크기, 줄 간격, 여백, 테마, 글꼴 변경.
- 설정 변경 후에도 문서 구조와 reference 연결이 유지되는지 확인.
- 마지막 읽던 위치가 같은 텍스트 근처로 복원되는지 확인.
- Android 볼륨 다운/업 페이지 이동 확인.
- iOS 대체 이동 컨트롤 확인.

### Reference와 Asset

- 그림/표/수식/각주/참고문헌 span 탭.
- 하단 시트 열기 확인.
- 전체 화면 열기, 확대, 이동 확인.
- reference span 위를 길게 누른 뒤 드래그했을 때 링크가 열리지 않고 텍스트 선택이 되는지 확인.

### OpenAI와 단어장

- OpenAI API key 입력과 보안 저장 확인.
- 단어 길게 누르기 후 문맥 기반 한국어 뜻 확인.
- 단어장 자동 저장 확인.
- 구/숙어 드래그 선택 후 선택적 단어장 저장 확인.
- 같은 논문 안에서 중복 저장이 막히는지 확인.
- 같은 표현이 다른 논문에서는 독립적으로 저장되는지 확인.
- 선택 문장 번역 확인.
- 선택 구간 요약 확인.
- 섹션 요약과 캐시 확인.

## MVP 제외 범위

- 사용자 계정.
- 기기 간 동기화.
- 서비스 소유 OpenAI key.
- 서버 측 OpenAI 프록시.
- 본문 안 그림 확장.
- 단어장 학습/퀴즈 시스템.
- 풍부한 커스텀 글꼴 카탈로그.
- iOS 볼륨키 페이지 이동 보장.
- 모든 PDF에 대한 완벽한 구조 추출.

## 성공 기준

- 사용자가 PDF를 가져와 모바일 친화적인 리플로우 형식으로 읽을 수 있다.
- 최초 변환은 시간이 걸릴 수 있지만, 같은 논문을 다시 열 때는 캐시된 패키지를 사용하고 구조가 예기치 않게 바뀌지 않는다.
- 핵심 뷰어 설정이 사용자가 요청한 카카오페이지식 조작과 맞는다.
- 그림/표/수식/각주/참고문헌 span을 본문에서 열 수 있다.
- 앱을 다시 열면 마지막 논문과 마지막 텍스트 위치 근처로 복원된다.
- Android에서 볼륨키로 페이지를 이동할 수 있다.
- 사용자의 OpenAI API key로 번역과 요약이 동작한다.
- 논문별 단어장이 단어와 여러 단어 표현을 중복 없이 저장한다.
